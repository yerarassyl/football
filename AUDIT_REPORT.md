# Аудит Air Arena

Дата аудита: 14 июля 2026 года.

## Область проверки

Air Arena — сервис подачи заявок на аренду футбольного поля. В проекте есть публичная форма бронирования, административная панель, аналитика, реферальные ссылки, Telegram-уведомления и Google Sheets через Google Apps Script. Регистрации клиентов, личного кабинета, товарного каталога, склада и пользовательской корзины нет; соответствующие ecommerce-сценарии неприменимы.

Технологии: Next.js 15 App Router, React 19, TypeScript, Google Sheets API/Google Apps Script, Telegram Bot API, Vercel. Постоянная пользовательская роль одна — администратор. Публичный пользователь может только создать заявку и проверить занятость.

## Сценарии и бизнес-логика

- Публичная цена пересчитывается на сервере, подмена `price/salePrice` клиентом не проходит: подтверждено.
- Конфликт по времени и пересекающимся четвертям реализован, но до исправления не был атомарным в Apps Script.
- Неавторизованные GET/PATCH/DELETE брони и PATCH настроек возвращают 401: подтверждено.
- Клиентских аккаунтов и доступа к «своим заказам» нет; вся клиентская информация доступна только admin API.
- Регистрация, восстановление пароля, профиль, каталог, склад и ecommerce-корзина отсутствуют и не тестируются как функции этого продукта.
- `npm audit --omit=dev`: 0 известных уязвимостей в 105 production dependencies на момент проверки.
- First Load JS по последней production-сборке: около 111 КБ для `/`, 119 КБ для `/admin`; это приемлемо для текущего объема.

## Ограничение визуальной проверки

Встроенный браузер Codex не запустился из-за ошибки записи служебных файлов среды. HTTP-маршруты, консоль dev-сервера, сборка, CSS breakpoints и DOM-разметка проверены; приложенные ранее мобильные скриншоты также учтены. Полноценный Lighthouse и интерактивный прогон всех семи viewport необходимо повторить после восстановления browser runtime.
